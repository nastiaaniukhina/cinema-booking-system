# Cinema Seat Booking — Hexagonal Architecture

## 1. Назва роботи

Система бронювання місця в кінотеатрі — Hexagonal Architecture

## 2. Опис use case

Дана система реалізує сценарій бронювання місця в кінотеатрі.
Користувач вводить номер місця, після чого система перевіряє, чи є воно доступним. Якщо місце вільне — бронювання виконується успішно. Якщо місце вже зайняте — система повідомляє про помилку.

## 3. Обрана архітектура

Hexagonal Architecture (Ports and Adapters)

## 4. Обґрунтування вибору

Обрана архітектура дозволяє чітко відокремити бізнес-логіку від зовнішніх залежностей (ввід/вивід, зберігання даних).
Це забезпечує кращу масштабованість, тестованість та гнучкість системи, оскільки зміни в інфраструктурі не впливають на основну логіку.

## 5. Структура проєкту

Проєкт має наступну структуру:

* domain/ — містить бізнес-сутності та доменну логіку
* application/ — містить сценарії використання (use case)
* ports/ — містить інтерфейси для взаємодії з зовнішніми компонентами
* adapters/ — містить реалізації інтерфейсів (робота з даними)
* main.cpp — точка входу в програму
* README.md — опис проєкту

## 6. Розподіл відповідальностей

* **domain logic:**
  Реалізована у файлі `domain/Seat.h`.
  Містить сутність Seat та бізнес-правило: неможливо забронювати вже зайняте місце.

* **use case:**
  Реалізований у файлі `application/BookSeatUseCase.h`.
  Відповідає за сценарій бронювання місця.

* **ports / interfaces:**
  Реалізовані у файлі `ports/SeatRepository.h`.
  Це абстракція доступу до даних.

* **adapters:**
  Реалізовані у файлі `adapters/InMemorySeatRepository.h`.
  Містять конкретну реалізацію збереження даних у пам’яті.

* **технічна реалізація:**
  Файл `main.cpp` відповідає за взаємодію з користувачем (ввід/вивід).

## 7. Інструкція запуску

### Варіант 1 (локально)

Скомпілювати та запустити програму:

g++ main.cpp -o app
./app

### Варіант 2 (онлайн)

Можна використати онлайн-компілятор:
https://www.onlinegdb.com/online_c++_compiler

Скопіювати код у редактор та натиснути Run.
